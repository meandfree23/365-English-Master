# Visual Logic v3.0 (Interactive Components & Web Visualization)

이 지침은 **데이터의 인터랙티브한 전달**과 **컴포넌트 중심의 현대적 디자인**을 구현하기 위한 최종 고도화 지침입니다. 

---

## 🏗️ 1. Interactive Component Gallery (웹 기반 시각화)

### 데이터 시각화 (Charts & Graphs)
- **Chart.js / Recharts**: 정적인 이미지를 넘어 마우스 오버(Hover) 시 세부 데이터가 표시되는 인터랙티브 차트 사용.
- **Progress Bar / Donut Chart**: 목표 달성률 및 비중을 실시간 애니메이션과 함께 시각화.

### 구조적 컴포넌트 (Structural Components)
- **Carousel (캐러셀)**: 여러 장의 이미지를 넘겨볼 수 있는 슬라이더 구현.
- **Tree View / Accordion**: 계층 구조가 복잡한 정보를 접었다 펴는 방식으로 계층화.
- **Matrix / Heatmap**: 복잡한 데이터 간의 상관관계를 색상의 농도로 객관적으로 시각화.

---

## 📐 2. UI/UX 디자인 규칙 (Design System Logic)

### 여백 및 배율 규칙 (Spacing System)
- **8px Grid System**: 모든 컴포넌트 간의 간격(Padding/Margin)은 8의 배수로 설정하여 전문적인 여백 관리.
- **Z-Index**: 레이어의 높낮이를 명확히 하여 정보의 전면과 후면 관계 시각화.

### 애니메이션 프로토콜 (Animation Protocol)
- **Fade-In / Stagger**: 요소들이 순차적으로 나타나게 하여 시선의 흐름 제어.
- **Smooth Transition**: 페이지 전환 시 0.3s-0.5s의 자연스러운 전환 효과 적용.
- **Hover Effects**: 인터랙티브 요소를 시각적으로 표시하여 사용자 클릭 유도.

---

## 🛠️ 3. 하이브리드 자동화 공정 연계

1. **Static Stage**: 파워포인트 내 '도형 병합-교차'를 통한 인포그래픽 정밀 가공.
2. **Interactive Stage**: 클로드 아티팩트(Artifacts)를 통한 웹 사이트 형태의 프젠테이션 렌더링.
3. **Hybrid Stage**: 웹 렌더링 화면을 스크린샷하거나 PNG로 변환하여 정적 파워포인트에 풍부한 시각 자료로 재배치.

---
🚀 **감수 지침**: 
Visual Logic Auditor는 각 컴포넌트가 **"정보의 논리적 구조를 인터랙티브하게 강화하는가?"**를 심사합니다. 모든 기술적 시각화는 사용자의 데이터 이해를 10배 더 빠르게 만드는 데 사용되어야 합니다.
